# บทที่ 5 สรุปผลและข้อเสนอแนะ

## 5.1 สรุปและอภิปรายผล

โครงงานนี้ได้ออกแบบและพัฒนา **ระบบควบคุมหลอดไฟอัจฉริยะผ่านโพรโทคอล MQTT** ที่ทำงานบนเครือข่าย
ภายในบ้าน โดยมีเซิร์ฟเวอร์เป็น **ศูนย์กลางความจริงเพียงหนึ่งเดียว (Single Source of Truth)** ระบบ
ประกอบด้วยเฟิร์มแวร์บนไมโครคอนโทรลเลอร์ ESP8266 ที่ควบคุมรีเลย์และรองรับการสั่งงานด้วยเสียง,
เซิร์ฟเวอร์ที่พัฒนาด้วย Express 5 และ MongoDB ทำหน้าที่รับ-ส่ง MQTT, REST API และต้นทาง SSE,
และหน้าเว็บที่พัฒนาด้วย Vue 3 สำหรับแสดงสถานะเรียลไทม์และสั่งงาน

เมื่อพิจารณาผลการดำเนินงานเทียบกับวัตถุประสงค์ในบทที่ 1 พบว่าระบบบรรลุวัตถุประสงค์ครบทุกข้อ ดังนี้

1. **ทำงานภายในเครือข่าย ไม่พึ่งคลาวด์** — ระบบทั้งหมดทำงานบน LAN ตอบสนองรวดเร็วและทำงานได้แม้
   อินเทอร์เน็ตขัดข้อง
2. **ประยุกต์ใช้ MQTT ได้เหมาะสม** — การเลือกใช้ QoS, retained message และ LWT ให้เหมาะกับลักษณะ
   ข้อมูลแต่ละประเภท (คำสั่ง vs. heartbeat vs. การตรวจจับออฟไลน์) ช่วยให้ระบบมีประสิทธิภาพและน่าเชื่อถือ
3. **แก้ปัญหาความไม่สอดคล้องของสถานะ** — หลัก Single Source of Truth ที่ให้ทุกการเปลี่ยนแปลงวิ่งผ่าน
   เส้นทางเดียวและกระจายแบบ broadcast แก้ปัญหาที่ระบุไว้ในบทที่ 1 ได้อย่างเป็นระบบ
4. **การจับคู่และตรวจจับสถานะมีความน่าเชื่อถือ** — กลไก Announce & Claim และการตรวจจับออฟไลน์
   หลายชั้นทำงานได้จริงตามที่ออกแบบ
5. **หน้าเว็บแสดงผลเรียลไทม์** — ทำได้ครบถ้วน

จุดเด่นสำคัญคือการออกแบบกลไกปรับสถานะให้ตรงกัน (drift reconciliation) ที่ทำงานหลายชั้นทั้งจาก ack,
timeout และ heartbeat ทำให้แม้คำสั่งสูญหายหรืออุปกรณ์รีบูตเอง ระบบก็ยังกลับมาสู่สถานะที่ถูกต้องได้เอง
โดยไม่ต้องให้ผู้ใช้แทรกแซง

## 5.2 ปัญหาและข้อจำกัด

**ปัญหาและอุปสรรคในการดำเนินงาน** `[ผู้จัดทำปรับตามประสบการณ์จริง]`

- การจัดการทรัพยากรที่จำกัดของ ESP8266 เมื่อใช้หลายไลบรารีร่วมกัน (MQTT, JSON, WiFiManager, UART)
- การจัดสรรขา GPIO ที่ปลอดภัยต่อการบูต เนื่องจากบางขาถูกใช้โดยโมดูลเสียง VC-02
- การออกแบบให้สถานะระหว่างฐานข้อมูลกับฮาร์ดแวร์ตรงกันในทุกกรณีขอบ (edge cases)

**ข้อจำกัดของระบบ**

1. ออกแบบสำหรับใช้งานภายในเครือข่ายเดียวกัน ยังไม่รองรับการเข้าถึงจากภายนอกผ่านอินเทอร์เน็ต
2. ควบคุมแบบเปิด/ปิดเป็นหลัก ยังไม่รองรับการหรี่แสง (dimming) แบบต่อเนื่อง
3. ยังไม่มีระบบยืนยันตัวตนผู้ใช้แบบหลายผู้ใช้และการกำหนดสิทธิ์
4. รองรับอุปกรณ์ประเภทหลอดไฟเป็นหลัก ยังไม่ครอบคลุมอุปกรณ์ IoT ประเภทอื่น

## 5.3 ข้อเสนอแนะและการพัฒนาต่อยอด

1. **เพิ่มการเข้าถึงจากภายนอก** — ผ่าน VPN, reverse proxy หรือ MQTT bridge เพื่อควบคุมจากนอกบ้าน
   โดยยังคงความปลอดภัย
2. **เพิ่มระบบยืนยันตัวตนและการกำหนดสิทธิ์** — รองรับผู้ใช้หลายคนและบทบาทที่ต่างกัน
3. **รองรับการหรี่แสงและอุปกรณ์ประเภทอื่น** — ขยายโครงสร้างข้อมูลและโพรโทคอลให้รองรับเซนเซอร์
   หรือสวิตช์ประเภทต่าง ๆ
4. **เพิ่มความปลอดภัยของ MQTT** — เปิดใช้การยืนยันตัวตน (username/password) และการเข้ารหัส TLS
5. **เพิ่มหน้าแสดงสถิติและการวิเคราะห์** — สรุปการใช้งานจากประวัติ device log ที่บันทึกไว้
6. **รองรับการขยายขนาด (scaling)** — เพิ่มการแบ่งช่อง (channel/room) ของ SSE เมื่อมีอุปกรณ์และผู้ใช้
   จำนวนมาก

โดยสรุป โครงงานนี้ได้พัฒนาระบบควบคุมหลอดไฟอัจฉริยะที่ทำงานได้จริงและสาธิตหลักการออกแบบระบบ
แบบกระจายที่เน้นความสอดคล้องของสถานะ ซึ่งสามารถนำไปต่อยอดและประยุกต์กับงาน IoT ในวงกว้างได้
